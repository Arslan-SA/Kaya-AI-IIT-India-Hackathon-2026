## The Problem

Once a wall is plastered, a slab is poured, or a pipe is buried — that work disappears.

No attribution. No evidence. No history.

Software solved this decades ago with version control. Physical construction still runs on WhatsApp photos and word of mouth.

The result: **$43M** average construction dispute value. **9%** of all project costs lost to rework. Zero accountability at the worker level.

---

## What Chisel Does

Chisel brings the commit-verify-merge workflow to jobsites.

```
worker completes work
      │
      ▼
  git chisel commit "Laid foundation reinforcement — Block C, Level 2"
  + photo/video evidence attached
      │
      ▼
  peers on-site signal: looks good ✓
      │
      ▼
  senior supervisor reviews remotely → approves
      │
      ▼
  work merged into project history — permanent, attributed, tamper-evident
```

Every piece of physical work becomes a **named, evidenced, reviewed, immutable record.**

---

## Core Features

| Feature | What it does |
|---|---|
| **Chisel Commit** | Workers log completed work as discrete units with photo/video proof |
| **Peer Signal** | On-site colleagues give lightweight advisory verification |
| **Senior Review** | Supervisors approve or flag work remotely from submitted evidence |
| **Issue Tracking** | Defects and rework logged as trackable issues — assigned, followed up, closed |
| **Permanent Record** | Verified work becomes part of an auditable project history |

## The Security Model

> *Chisel's entire value is "this record is real and can't be faked."*

- **Append-only audit trail** — commits cannot be edited or deleted post-verification
- **Cryptographic integrity** — hash-chained records, tamper-evident by design
- **Media authenticity** — metadata verification to prevent backdated photo submissions
- **Identity verification** — role-based access, no self-approval of own commits
- **Collusion detection** — AI flags statistically anomalous peer-approval patterns
